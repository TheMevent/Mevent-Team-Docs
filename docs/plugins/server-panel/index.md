# ServerPanel

## Что делает этот плагин?

**ServerPanel** позволяет вашему серверу иметь красивое полноэкранное информационное меню, которое открывается у игроков при входе на сервер или по команде. В меню можно разместить правила сервера, информацию о рейтах, карту, контакты администрации, ссылки на Discord и многое другое. Администраторы могут редактировать содержимое прямо в игре через встроенный визуальный редактор без перезагрузки плагина.

## Ключевые возможности

- Игроки видят полноэкранное меню с информацией о сервере при входе или по команде
- Игроки могут переключаться между категориями и страницами меню
- Игроки видят свой баланс экономики (Economics, ServerRewards, BankSystem и др.) в шапке меню
- Администраторы могут редактировать меню прямо в игре (добавлять текст, изображения, кнопки)
- Администраторы могут создавать неограниченное количество категорий и страниц
- Поддержка всплывающих окон (Pop-Ups)
- Поддержка локализации -- меню на разных языках для разных игроков
- Встроенный установщик шаблонов для быстрого старта
- Блокировка открытия меню при рейд-блоке, билд-блоке или боевом блоке
- Автоматический показ меню при подключении игрока
- Интеграция с другими плагинами через API

## Требования

| Зависимость | Обязательность | Описание |
|---|---|---|
| **ImageLibrary** | Обязательно | Загрузка и отображение изображений в интерфейсе |
| **ServerPanel Installer** | Рекомендуется | Быстрая установка готовых шаблонов меню |
| **ServerPanel PopUps** | По желанию | Всплывающие окна внутри меню |
| **ServerPanel Migrations** | По желанию | Автоматическая миграция данных при обновлении |
| **Notify** / **UINotify** | По желанию | Отображение уведомлений через систему Notify |
| **NoEscape** | По желанию | Блокировка меню при рейд-блоке и боевом блоке |
| Oxide / uMod / Carbon | Обязательно | Совместим с Oxide, uMod и Carbon |

## Быстрый старт

1. Установите **ImageLibrary** на сервер.
2. Скопируйте файл `ServerPanel.cs` и дополнительные файлы (`ServerPanelInstaller.cs`, `ServerPanelPopUps.cs`, `ServerPanelMigrations.cs`) в папку `oxide/plugins/`.
3. Перезагрузите сервер.
4. Зайдите на сервер и используйте установщик для выбора шаблона:
 ```
 serverpanelinstaller.open
 ```
5. Выберите понравившийся шаблон и установите его.
6. Настройте команды открытия меню через данные категорий.

:::tip Совет
Для первой установки рекомендуется использовать встроенный установщик шаблонов -- это значительно упростит настройку.
:::

## Права доступа

| Право | Кому нужно | Что открывает |
|---|---|---|
| `serverpanel.edit` | Администраторам | Доступ к встроенному визуальному редактору меню. Позволяет добавлять, удалять и редактировать элементы, категории и страницы прямо в игре |
| `serverpanelinstaller.admin` | Администраторам | Доступ к установщику шаблонов |
| `serverpanelpopups.edit` | Администраторам | Редактирование всплывающих окон |
| Права категорий (настраиваются) | Игрокам | Доступ к конкретным категориям меню (задаются в настройках каждой категории) |

## Команды игроков

Команды для открытия меню настраиваются администратором для каждой категории. По умолчанию используются команды, указанные при создании категорий.

| Команда | Описание | Пример |
|---|---|---|
| `/(команда категории)` | Открывает меню на указанной категории | `/menu`, `/info`, `/rules`, `/shop` и др. |

:::info Примечание
Конкретные команды зависят от того, как администратор настроил категории. Каждая категория может иметь одну или несколько команд для открытия.
:::

## Команды администратора

| Команда | Описание | Пример |
|---|---|---|
| `serverpanelinstaller.open` | Открыть установщик шаблонов (для первой настройки) | `serverpanelinstaller.open` |
| `sp.migrations list` | Посмотреть доступные миграции данных | `sp.migrations list` |
| `sp.migrations run [имя]` | Запустить миграцию данных | `sp.migrations run all` |

### Режим редактирования

Игроки с правом `serverpanel.edit` видят специальные кнопки в интерфейсе для входа в режим редактирования:

- **Редактирование страниц** -- добавление, удаление, клонирование и перемещение страниц
- **Редактирование элементов** -- добавление текста, изображений, кнопок на страницу
- **Редактирование заголовка** -- изменение полей в шапке меню
- **Редактирование категорий** -- создание, удаление и настройка категорий

## Устранение неполадок

### Меню не открывается при вводе команды

**Симптомы:** Игрок вводит команду, но меню не появляется.

**Причина:** Команда не привязана к категории, или у игрока нет права доступа.

**Решение:** Проверьте, что команда настроена в данных категории. Убедитесь, что категория включена и у игрока есть необходимое право.

---

### Ошибка "Plugin is not initialized"

**Симптомы:** При попытке открыть меню появляется сообщение об ошибке.

**Причина:** Данные плагина не загружены. Возможно, требуется начальная установка шаблона.

**Решение:** Используйте установщик шаблонов: `serverpanelinstaller.open` (требуется право `serverpanelinstaller.admin`).

---

### Изображения не загружаются

**Симптомы:** Вместо изображений отображаются пустые области.

**Причина:** ImageLibrary не установлена или не успела загрузить изображения.

**Решение:** Убедитесь, что ImageLibrary установлена и загружена. Перезагрузите ServerPanel. Проверьте доступность ссылок на изображения.

---

### Меню открывается, но содержимое пустое

**Симптомы:** Видно фон меню, но нет текста и элементов.

**Причина:** Требуется миграция данных после обновления плагина.

**Решение:** Установите `ServerPanelMigrations.cs` и дождитесь автоматической миграции, или запустите её вручную: `sp.migrations run all`.

---

### Меню не открывается из-за рейд-блока / билд-блока

**Симптомы:** Игрок получает сообщение о блокировке при попытке открыть меню.

**Причина:** Включена блокировка открытия в зоне строительства или во время рейд-блока.

**Решение:** Если вы хотите отключить эти блокировки, измените соответствующие параметры в конфигурации (секция "Block Settings").

---

### Меню не показывается при подключении игрока

**Симптомы:** Игроки не видят меню при входе на сервер.

**Причина:** Функция автоматического показа отключена или нет доступных категорий.

**Решение:** Убедитесь, что параметр "Show menu every time player connects to server?" установлен в `true`. Проверьте, что хотя бы одна категория включена и видима.
