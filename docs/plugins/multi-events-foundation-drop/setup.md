# Установка и настройка

Это руководство проведёт вас через процесс установки режима Foundation Drop для системы MultiEvents.

## Системные требования

- Сервер Rust с Oxide / uMod / Carbon
- Установленный плагин **MultiEvents** (обязательная зависимость)
- Установленный плагин **ImageLibrary** (обязательная зависимость)

## Шаг 1: Установите зависимости

### MultiEvents (обязательно)

Плагин Foundation Drop является режимом расширения и требует основной плагин MultiEvents.

1. Установите **MultiEvents** согласно его документации.
2. Убедитесь, что плагин загружен без ошибок.

### ImageLibrary (обязательно)

1. Скачайте **ImageLibrary** с [umod.org](https://umod.org/plugins/image-library).
2. Поместите файл `ImageLibrary.cs` в папку `oxide/plugins/`.

:::warning Важно
Без MultiEvents и ImageLibrary плагин Foundation Drop **не будет работать**.
:::

## Шаг 2: Установите плагин

1. Скопируйте файл `MultiEventsFoundationDrop.cs` в папку `oxide/plugins/` вашего сервера.
2. Плагин загрузится автоматически и зарегистрируется в MultiEvents.
3. Проверьте консоль на наличие ошибок.

Файл конфигурации будет создан автоматически:
```
oxide/config/MultiEventsFoundationDrop.json
```

## Шаг 3: Настройте права доступа

Выдайте права администраторам:

```
oxide.grant group admin MultiEventsFoundationDrop.start
oxide.grant group admin MultiEventsFoundationDrop.stop
```

## Шаг 4: Настройте арену

В конфигурации настройте:

- **Minimal players to start event** -- минимальное количество игроков (не менее 2)
- **Arena Settings / Size (matrix NxN)** -- размер платформы (например, 10 = сетка 10x10 фундаментов)

## Шаг 5: Проверьте работу

Запустите ивент:

```
foundationdrop.start
```

Начнётся фаза голосования, во время которой игроки могут присоединиться. После набора достаточного количества участников игра начнётся.

## Шаг 6: Настройте конфигурацию (по желанию)

Откройте файл `oxide/config/MultiEventsFoundationDrop.json` для настройки:

- **Минимальное количество игроков** -- порог для старта
- **Размер арены** -- размер сетки фундаментов
- **Задержки** -- время голосования и ожидания
- **Настройки финальной битвы** -- радиация, оружие, кит
- **Награды** -- настройка наград для победителя
- **Разрешённые команды** -- какие команды можно использовать во время ивента

Подробнее в разделе [Конфигурация](./configuration).

## Шаг 7: Перезагрузите плагин

После изменения конфигурации:

```
oxide.reload MultiEventsFoundationDrop
```

:::tip Совет
Рекомендуется сначала протестировать ивент с небольшой ареной (5x5) и минимальным количеством игроков (2), чтобы убедиться в корректной работе.
:::
