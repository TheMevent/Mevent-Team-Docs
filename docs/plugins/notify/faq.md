# Часто задаваемые вопросы (FAQ)

## 1. Как добавить свой тип уведомления?

Откройте файл конфигурации `oxide/config/Notify.json` и в секции `"Notifications (type - settings)"` добавьте новую запись с уникальным числовым идентификатором. Скопируйте структуру существующего типа (например, `"0"`) и измените цвета, иконку и другие параметры под свои нужды.

**Пример:** чтобы добавить тип 2 с зелёным цветом:

```json
"2": {
 "Enabled": true,
 "Background Color": { "HEX": "#000000", "Opacity (0 - 100)": 98.0 },
 "Enable Gradient?": true,
 "Gradient Color": { "HEX": "#0FF542", "Opacity (0 - 100)": 35.0 },
 "Icon Color": { "HEX": "#0FF542", "Opacity (0 - 100)": 100.0 },
 "Icon Text": "i",
 "Title Key (lang)": "Notification",
 ...
}
```

Не забудьте перезагрузить плагин после изменений: `oxide.reload Notify`.

---

## 2. Можно ли сделать уведомление кликабельным?

Да. В настройках типа уведомления включите параметр `"Use command": true` и укажите команду в поле `"Command"`. Например, `"Command": "shop"` откроет магазин при клике на уведомление. Если включён параметр `"Close after using the command?"`, уведомление закроется после клика.

---

## 3. Почему некоторые игроки видят сообщения в чате вместо графических уведомлений?

Это происходит, если у игрока нет права `notify.see`. Плагин автоматически отправляет текстовое сообщение в чат как запасной вариант. Это поведение можно отключить, установив параметр `"Send text message to chat if player doesn't have notification permission"` в `false`.

---

## 4. Как изменить позицию уведомлений на экране?

В конфигурации измените параметр `"Display notifications on the top right?"`:
- `true` -- уведомления в правом верхнем углу экрана (по умолчанию)
- `false` -- уведомления в левом нижнем углу экрана

Для тонкой настройки положения используйте параметры `"X Margin"`, `"Y Margin"` и `"Y Indent"`.

---

## 5. Как другие плагины могут отправлять уведомления через Notify?

Другие плагины используют API-вызов для отправки уведомлений. Для этого плагин должен объявить ссылку на Notify и вызывать специальную функцию с указанием игрока, типа уведомления и текста сообщения. Многие плагины от Mevent уже имеют встроенную поддержку Notify -- достаточно включить опцию `"Work with Notify?"` в их конфигурации.

---

## 6. Можно ли использовать свои изображения для иконок уведомлений?

Да. В настройках типа уведомления в секции "Image Settings" установите `"Enabled": true` и укажите URL изображения в поле `"Image"`. Изображение будет загружено через ImageLibrary и отображено вместо текстовой иконки. Рекомендуется использовать изображения в формате PNG небольшого размера.

---

## 7. Уведомления перекрывают мини-карту или другие элементы интерфейса. Как это исправить?

Попробуйте следующее:
1. Измените параметр `"Display type"` с `"OverlayNonScaled"` на `"Hud"`.
2. Увеличьте значение `"Y Indent"` (более отрицательное), чтобы опустить уведомления ниже.
3. Увеличьте `"X Margin"`, чтобы отодвинуть уведомления от края экрана.
4. Переместите уведомления в другой угол, изменив `"Display notifications on the top right?"` на `false`.

---

## 8. Сколько типов уведомлений можно создать?

Ограничений нет. Вы можете создать любое количество типов уведомлений, присвоив каждому уникальный числовой идентификатор. Каждый тип может иметь свои цвета, иконки, звуковые эффекты, размеры и поведение.

---

## 9. Как отключить звуковые эффекты уведомлений?

В настройках нужного типа уведомления очистите поле `"Sound Effect (empty - disable)"`, установив его в пустую строку `""`. Это отключит звук для данного типа.

---

## 10. Можно ли задать разное время отображения для разных типов уведомлений?

Да. В настройках типа уведомления установите `"Use custom cooldown": true` и укажите нужное время в секундах в поле `"Cooldown"`. Если индивидуальное время не задано, используется глобальный параметр `"Notify Cooldown"`.
