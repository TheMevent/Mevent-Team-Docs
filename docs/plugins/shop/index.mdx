# Shop

## Что делает этот плагин?

**Shop** позволяет вашему серверу добавить полноценный магазин с красивым интерфейсом, где игроки покупают и продают предметы за внутриигровую валюту. Плагин поддерживает категории товаров, NPC-продавцов, привязку к торговым автоматам, систему скидок, корзину покупок, избранные товары, лимиты покупок, несколько валют и переводы денег между игроками.

## Ключевые возможности

- Игроки могут покупать и продавать предметы через удобный графический интерфейс
- Игроки могут переводить деньги друг другу
- Игроки могут добавлять товары в избранное и использовать повторные покупки
- Администраторы могут настраивать категории, цены и лимиты прямо в игре
- Администраторы могут привязывать магазин к NPC и торговым автоматам
- Поддержка нескольких валют одновременно (Economics, ServerRewards, предметы)
- Система скидок на основе прав доступа
- Блокировка магазина после вайпа, респавна или во время дуэлей
- Логирование покупок и продаж в файл, консоль и Discord
- Шаблоны оформления магазина с установкой через команду

## Требования

- **Oxide** или **Carbon** последней версии
- **ImageLibrary** -- обязательно для отображения интерфейса
- **Notify** или **UINotify** -- рекомендуется для уведомлений (необязательно)
- **Economics** или другой плагин экономики -- для основной валюты (необязательно, можно использовать предметы)
- **LangAPI** -- для мультиязычной поддержки (необязательно)

## Быстрый старт

1. Установите **ImageLibrary** на сервер.
2. Поместите файл `Shop.cs` в папку `oxide/plugins`.
3. Перезагрузите сервер или выполните `o.reload Shop`.
4. Выполните команду `/shop.install` в чате (с правами администратора) для установки шаблона магазина.
5. Настройте конфигурацию и предметы магазина.
6. Игроки могут использовать команду `/shop` для открытия магазина.

## Права доступа

| Право | Кому нужно | Что открывает |
|---|---|---|
| `shop.use` (настраивается) | Игрокам | Доступ к интерфейсу магазина (если задано в конфиге) |
| `shop.admin` | Администраторам | Полный доступ к редактированию магазина, установке шаблонов |
| `shop.free` | Особым игрокам | Бесплатные покупки (обход цен) |
| `shop.setvm` | Администраторам | Привязка магазина к торговым автоматам |
| `shop.setnpc` | Администраторам | Привязка магазина к NPC |
| `shop.bypass.dlc` | Игрокам | Обход ограничений на DLC-предметы |
| `shop.default` | Обычным игрокам | Скидка 0% (настраивается) |
| `shop.vip` | VIP-игрокам | Скидка 10% (настраивается) |
| `shop.buyagain` (настраивается) | Игрокам | Доступ к функции повторной покупки |
| `shop.usenpc` (настраивается) | Игрокам | Доступ к определённым NPC-магазинам |

## Команды игроков

| Команда | Описание | Пример |
|---|---|---|
| `/shop` | Открыть интерфейс магазина | `/shop` |
| `/shops` | Альтернативная команда для открытия магазина | `/shops` |

:::tip Примечание
Команды настраиваются в конфиге. По умолчанию доступны `/shop` и `/shops`.
:::

## Команды администратора

| Команда | Описание | Пример |
|---|---|---|
| `/shop.install` | Установить шаблон магазина | `/shop.install` |
| `/shop.setvm` | Привязать магазин к торговому автомату (смотрите на него) | `/shop.setvm` |
| `/shop.setnpc` | Привязать магазин к NPC (смотрите на него) | `/shop.setnpc` |

## Устранение неполадок

### Магазин не открывается
**Симптом:** Команда `/shop` не вызывает интерфейс.
**Причина:** Не установлен ImageLibrary или шаблон магазина не инициализирован.
**Решение:** Установите ImageLibrary и выполните `/shop.install` с правами администратора.

### Сообщение "Shop is initializing"
**Симптом:** При попытке открыть магазин появляется сообщение о инициализации.
**Причина:** Плагин ещё загружает изображения товаров.
**Решение:** Подождите завершения загрузки (обычно 1-3 минуты после старта сервера).

### Игрок не может купить DLC-предмет
**Симптом:** При покупке DLC-предмета появляется ошибка.
**Причина:** У игрока нет соответствующего DLC.
**Решение:** Выдайте игроку право `shop.bypass.dlc` или уберите DLC-предметы из магазина.

### Некорректные цены после обновления
**Симптом:** Цены отображаются неправильно или сброшены.
**Причина:** Обновление плагина изменило структуру данных.
**Решение:** Проверьте данные магазина и пересоздайте при необходимости с помощью `/shop.install`.

### Переводы денег не работают
**Симптом:** Игроки не могут переводить деньги друг другу.
**Причина:** Функция переводов отключена в конфиге.
**Решение:** Установите `Enable money transfers between players?` в `true`.

### Магазин заблокирован после респавна
**Симптом:** Игрок не может открыть магазин сразу после смерти.
**Причина:** Включена блокировка после респавна.
**Решение:** Это нормальное поведение. Подождите указанное время (по умолчанию 60 секунд) или отключите `Respawn Block` в конфиге.
