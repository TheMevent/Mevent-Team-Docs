# Notify

## Что делает этот плагин?

**Notify** позволяет вашему серверу показывать игрокам красивые всплывающие уведомления прямо на экране. Уведомления появляются в углу экрана, сопровождаются звуковыми эффектами и плавной анимацией. Другие плагины могут использовать Notify как единую систему уведомлений, чтобы все сообщения на сервере выглядели в одном стиле.

## Ключевые возможности

- Игроки видят стильные всплывающие уведомления вместо стандартных сообщений в чате
- Игроки без специального права получают сообщения в чат (настраивается)
- Администраторы могут отправлять уведомления конкретному игроку, себе или всем игрокам на сервере
- Полная настройка внешнего вида: цвета, иконки, фоновые изображения, градиенты
- Поддержка нескольких типов уведомлений (информация, ошибка, награда и другие)
- Интерактивные уведомления с кнопками, выполняющими команды по клику
- Звуковые эффекты при появлении уведомлений
- Другие плагины могут вызывать уведомления через встроенный API

## Требования

| Зависимость | Обязательность | Описание |
|---|---|---|
| **ImageLibrary** | Обязательно | Необходима для загрузки и отображения пользовательских изображений в уведомлениях |
| Oxide / uMod / Carbon | Обязательно | Совместим с Oxide, uMod и Carbon |

## Быстрый старт

1. Установите плагин **ImageLibrary**, если он ещё не установлен.
2. Скопируйте файл `Notify.cs` в папку `oxide/plugins/` вашего сервера.
3. Перезагрузите сервер или выполните команду `oxide.reload Notify`.
4. Выдайте право на просмотр уведомлений всем игрокам:
 ```
 oxide.grant group default notify.see
 ```
5. Проверьте работу, отправив тестовое уведомление:
 ```
 notify.show 0 Тестовое уведомление!
 ```

## Права доступа

| Право | Кому нужно | Что открывает |
|---|---|---|
| `notify.see` | Всем игрокам | Позволяет видеть графические уведомления на экране. Без этого права игрок получит текстовое сообщение в чат (если включено в настройках) |
| `notify.notify` | Администраторам | Позволяет использовать команду отправки уведомления самому себе |
| `notify.player` | Администраторам | Позволяет отправлять уведомления конкретному игроку по SteamID |
| `notify.allplayer` | Администраторам | Позволяет отправлять уведомления всем игрокам на сервере одновременно |

## Команды игроков

У этого плагина нет команд для обычных игроков. Игроки автоматически видят уведомления, которые им отправляют другие плагины или администраторы.

## Команды администратора

| Команда | Описание | Пример |
|---|---|---|
| `notify.show [тип] [сообщение]` | Отправить уведомление самому себе | `notify.show 0 Сервер перезагрузится через 5 минут` |
| `notify.player [steamid/имя] [тип] [сообщение]` | Отправить уведомление конкретному игроку | `notify.player 76561198012345678 1 Вы нарушили правила!` |
| `notify.allplayers [тип] [сообщение]` | Отправить уведомление всем игрокам | `notify.allplayers 0 Вайп через 1 час!` |

### Типы уведомлений по умолчанию

| Тип | Описание | Цвет иконки |
|---|---|---|
| `0` | Информационное уведомление | Синий (#4B68FF) |
| `1` | Уведомление об ошибке | Красный (#FF6060) |

:::tip Совет
Вы можете создавать собственные типы уведомлений, добавляя новые записи в конфигурацию с уникальными числовыми идентификаторами.
:::

## Устранение неполадок

### Уведомления не отображаются

**Симптомы:** Игроки не видят графических уведомлений, но сообщения приходят в чат.

**Причина:** У игроков не выдано право `notify.see`.

**Решение:** Выполните команду: `oxide.grant group default notify.see`

---

### Ошибка "IMAGE LIBRARY IS NOT INSTALLED!"

**Симптомы:** В консоли сервера появляется ошибка при загрузке плагина.

**Причина:** Плагин ImageLibrary не установлен или не загружен.

**Решение:** Скачайте и установите ImageLibrary в папку `oxide/plugins/`.

---

### Уведомления перекрывают другие элементы интерфейса

**Симптомы:** Уведомления появляются поверх важных элементов интерфейса.

**Причина:** Неправильная настройка позиции или типа отображения.

**Решение:** Измените параметр "Display type" в конфигурации на `Hud` вместо `OverlayNonScaled`, или измените положение уведомлений, отключив параметр "Display notifications on the top right".

---

### Уведомления исчезают слишком быстро или слишком медленно

**Симптомы:** Игроки не успевают прочитать уведомление, или оно висит слишком долго.

**Причина:** Значение параметра "Notify Cooldown" не соответствует вашим предпочтениям.

**Решение:** Измените значение "Notify Cooldown" в конфигурации (в секундах). По умолчанию: 10 секунд.

---

### Слишком много уведомлений на экране одновременно

**Симптомы:** Экран игрока заполнен уведомлениями, мешающими игре.

**Причина:** Высокий лимит одновременных уведомлений или частая отправка.

**Решение:** Уменьшите значение "Max Notifications On Screen" в конфигурации. По умолчанию: 10.

---

### Уведомления без фонового изображения выглядят неправильно

**Симптомы:** Фоновые изображения не загружаются, уведомления отображаются с ошибками.

**Причина:** Ссылки на изображения недоступны или ImageLibrary не успела их загрузить.

**Решение:** Проверьте доступность ссылок на изображения в конфигурации. Перезагрузите плагин после запуска ImageLibrary. Убедитесь, что ваш сервер имеет доступ к интернету.
