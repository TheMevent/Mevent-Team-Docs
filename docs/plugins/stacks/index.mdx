# Stacks

## Что делает этот плагин?

**Stacks** позволяет вашему серверу гибко управлять размерами стаков (максимальным количеством предметов в одной ячейке) для каждого предмета в игре. Вы можете устанавливать индивидуальные размеры стаков для любых предметов, а также настраивать множители для конкретных контейнеров (сундуки, печи, шкафы и т.д.). Плагин предоставляет удобный графический интерфейс для администраторов.

## Ключевые возможности

- Индивидуальная настройка размера стака для каждого предмета в игре
- Настройка множителей для контейнеров (сундуки, печи, холодильники и т.д.)
- Удобный графический интерфейс (GUI) с поиском и категориями
- Изменение множителей контейнеров при помощи киянки
- Блокировка отдельных предметов и скинов от изменения стаков
- Защита от "оверстака" (переполнения стака при перемещении)
- Совместимость с FurnaceSplitter, ImageLibrary, Notify, UINotify и LangAPI
- Консольные команды для управления стаками без GUI

## Требования

| Зависимость | Обязательна? | Описание |
| -------------- | ------------ | --------------------------------------- |
| ImageLibrary | Да | Требуется для отображения изображений в интерфейсе |
| Notify / UINotify | Нет | Опционально, для отправки уведомлений |
| LangAPI | Нет | Опционально, для локализации имён предметов |
| FurnaceSplitter| Нет | Учитывается при перемещении предметов в печи |

## Быстрый старт

1. Установите плагин **ImageLibrary** на ваш сервер (обязательная зависимость).
2. Поместите файл плагина в папку `oxide/plugins` вашего сервера.
3. Перезагрузите сервер или выполните `oxide.reload Stacks`.
4. Выдайте себе право доступа: `oxide.grant user ваш_steamid stacks.admin`.
5. Введите в чат `/stacks` для открытия панели управления.

## Права доступа

| Право | Описание |
| ---------------- | --------------------------------------------- |
| `stacks.admin` | Полный доступ к панели управления и командам |

## Команды игроков

У данного плагина нет команд, доступных обычным игрокам. Все команды предназначены для администраторов.

## Команды администратора

### Чат-команды

| Команда | Описание |
| ------------------------------------------------ | ------------------------------------------------- |
| `/stacks` | Открывает графическую панель управления стаками |
| `/stacks sethandstack [размер]` | Устанавливает размер стака для предмета в руке |
| `/stacks setstack [shortname] [размер]` | Устанавливает размер стака для указанного предмета |

**Примеры:**

```
/stacks
/stacks sethandstack 10000
/stacks setstack wood 50000
```

### Консольные команды

| Команда | Описание |
| ----------------------------------------------------- | ------------------------------------------- |
| `stacks.manage stack set [shortname] [размер]` | Устанавливает размер стака (из консоли) |
| `stacks.manage stack get [shortname]` | Показывает текущий размер стака предмета |
| `stacks.manage stack get all` | Показывает размеры стаков всех предметов |

**Примеры:**

```
stacks.manage stack set wood 50000
stacks.manage stack get wood
stacks.manage stack get all
```

## Устранение неполадок

| Проблема | Решение |
| -------- | ------- |
| Плагин не работает, отображается ошибка | Убедитесь, что плагин **ImageLibrary** установлен и загружен корректно |
| Нет доступа к панели управления | Проверьте, что у вас выдано право `stacks.admin` |
| Предметы не стакаются с новым размером | Убедитесь, что предмет или его скин не добавлен в список блокировки (Blocked List) в конфигурации |
| Стаки не применяются в определённых контейнерах | Проверьте множитель контейнера в панели управления (вкладка "Container") |
| Предметы теряются при перемещении между контейнерами | Включите опцию запрета оверстака в конфигурации (`Disallow overstack items`) |
| Изменение стака через киянку не работает | Убедитесь, что опция "Changing multiplies in containers using a hammer" включена в конфигурации |
