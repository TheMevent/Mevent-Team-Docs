# Frequently Asked Questions (FAQ)

## 1. How do I add a new category to the menu?

Enter edit mode (requires `serverpanel.edit` permission), click the mode toggle button in the category area, then click the add category button. Set the name, open command, and other settings. All changes save automatically.

---

## 2. How do I set up a command to open the menu?

Commands are tied to categories. In edit mode, open the settings for the category you want and add one or more commands. For example, if you want players to open rules with `/rules`, add `rules` to the commands list for the "Rules" category. The command gets registered automatically after saving.

---

## 3. How do I restrict access to a specific menu category?

In the category settings, set an access permission (e.g., `serverpanel.vip`). Then grant that permission to the players or groups who should have access:
```
oxide.grant group vip serverpanel.vip
```
Players without the permission won't see the category in the menu.

---

## 4. How do I add another economy plugin's balance to the menu header?

Open `oxide/config/ServerPanel.json` and add a new entry in the `"Economy Header Fields"` section. Specify the plugin name, function names for checking and modifying the balance, and a unique update key. Any economy plugin that has get/add/remove balance functions is supported. You can also use in-game items (like scrap) as currency by choosing the `Item` type.

---

## 5. Can I use ServerPanel with other plugins?

Yes. ServerPanel supports integration with other plugins in two ways:

1. **"Plugin" page type** -- the page content is generated by another plugin. This lets you embed stats or leaderboards directly in the menu.
2. **Buttons with commands** -- elements on a page can execute any command on click, opening other plugin interfaces.

ServerPanel also integrates with economy plugins (Economics, ServerRewards, BankSystem, IQEconomic), notification plugins (Notify, UINotify), and block plugins (NoEscape).

---

## 6. How do I transfer a configured menu to another server?

Copy these to the new server:
1. Config file: `oxide/config/ServerPanel.json`
2. Data folder: `oxide/data/ServerPanel/` (contains Categories.json, Template.json, HeaderFields.json, Localization.json)
3. All plugin files (ServerPanel.cs and extras)

Reload the plugin on the new server after copying. All images will be automatically loaded through ImageLibrary.

---

## 7. What is offline image mode and when should I use it?

Offline image mode (`"Enable Offline Image Mode": true`) loads images from a local folder `oxide/data/MeventImages/` instead of downloading them from URLs. This is useful for:
- Servers with unstable or limited internet
- Faster image loading
- Reducing dependency on external image hosts

Place your images in the folder and use filenames instead of URLs.

---

## 8. What happens when I update the plugin?

The plugin checks the config and data format version on update. If the data format changed, you'll need a migration:

1. Install `ServerPanelMigrations.cs`.
2. Migration runs automatically.
3. If it doesn't run on its own, trigger it manually: `sp.migrations run all`.
4. After a successful migration, the menu keeps working with the updated data format.

:::warning Important
Always back up `oxide/data/ServerPanel/` before updating the plugin.
:::

---

## 9. How do I disable the auto-open menu on join?

In `oxide/config/ServerPanel.json`, set:
```json
"Auto-Open Settings": {
 "Show menu every time player connects to server?": false
}
```
After this, the menu will only open via command.

---

## 10. How do I add video to the menu?

In edit mode, create a button element that plays video on click. Specify the video URL and link it to the playback command. The video plays through the game's built-in video player.

---

## 11. The menu lags with lots of elements. How do I optimize?

Tips for better performance:
- Split content across multiple pages instead of one long page
- Use reasonably sized images (no larger than 512x512 pixels)
- Increase "Cooldown between actions" to 0.3-0.5 seconds
- Reduce the number of economy fields in the header if you don't need them all
- Disable unused economy fields in the header

---

## 12. How do I make the menu multi-language?

ServerPanel supports localization. The plugin uses Oxide's standard lang file system. Category names and element text can be localized through `oxide/lang/*/ServerPanel.json`. Players see the menu in their language automatically when a translation is available.
