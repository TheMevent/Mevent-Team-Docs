# Vehicle Buy

## Что делает этот плагин?

**Vehicle Buy** позволяет вашему серверу добавить полноценный магазин транспорта, в котором игроки могут приобретать любые виды техники -- от миникоптеров и лодок до модульных автомобилей, лошадей и велосипедов. Плагин поддерживает покупку через графический интерфейс, торговые автоматы в мирных городах, а также отзыв и подбор купленного транспорта.

## Ключевые возможности

- Покупка более 19 типов транспорта через красивый UI
- Торговые автоматы в Bandit Camp, Outpost и Fishing Village
- Система отзыва транспорта (recall) -- верните машину к себе
- Система подбора транспорта (pickup) -- заберите технику в инвентарь
- Подбор молотком или командой (настраивается для каждого транспорта)
- Поддержка трёх валют: Economics, Server Rewards, внутриигровые предметы
- Кулдаун на покупку и лимит покупок для каждого транспорта
- Автоматическая установка деталей двигателя для модульных машин
- Защита от урона для купленного транспорта (опционально)

## Требования

| Зависимость | Обязательна | Описание |
|---|---|---|
| KaruzaEntitiesCommon | Да | Общая библиотека для работы с сущностями |
| ImageLibrary | Да | Загрузка и отображение изображений в интерфейсе |
| Economics | Нет | Плагин экономики (один из вариантов валюты) |
| ServerRewards | Нет | Плагин наград сервера (один из вариантов валюты) |
| BankSystem | Нет | Банковская система (один из вариантов валюты) |
| Notify / UINotify | Нет | Уведомления для игроков |

## Быстрый старт

1. Установите плагин `VehicleBuy.cs` и обязательные зависимости в папку `oxide/plugins`.
2. Перезагрузите сервер или выполните `oxide.reload VehicleBuy`.
3. Отредактируйте конфигурацию `oxide/config/VehicleBuy.json`.
4. Назначьте игрокам право `vehiclebuy.use` через консоль:
 ```
 oxide.grant group default vehiclebuy.use
 ```
5. Игроки могут ввести команду покупки (настраивается) или использовать торговые автоматы в мирных зонах.

## Права доступа

| Право | Описание |
|---|---|
| `vehiclebuy.use` | Разрешение на использование магазина транспорта |
| `vehiclebuy.free` | Бесплатная покупка транспорта |
| `vehiclebuy.pickup` | Разрешение на подбор транспорта |
| `vehiclebuy.recall` | Разрешение на отзыв транспорта |
| Настраиваемые (напр. `vehiclebuy.vip`) | Индивидуальные права для конкретных видов транспорта |

## Команды игроков

| Команда | Описание |
|---|---|
| `/vehiclebuy` | Открыть магазин транспорта |
| `/vb` | Открыть магазин транспорта (краткая команда) |
| `/vehicle` | Открыть магазин транспорта (альтернативная команда) |
| `/callback <название>` | Отозвать купленный транспорт к себе |
| `/pickup` | Подобрать транспорт рядом с игроком |

**Примеры:**
```
/vehiclebuy
/callback copter
/pickup
```

## Команды администратора

Администратор управляет плагином через конфигурацию и консольные команды Oxide для выдачи прав:

```
oxide.grant user <SteamID> vehiclebuy.use
oxide.grant user <SteamID> vehiclebuy.free
oxide.grant user <SteamID> vehiclebuy.pickup
oxide.grant user <SteamID> vehiclebuy.recall
```

## Устранение неполадок

| Проблема | Решение |
|---|---|
| Магазин не открывается | Проверьте, что у игрока есть право `vehiclebuy.use` и установлен **ImageLibrary** |
| Транспорт не появляется после покупки | Убедитесь, что рядом достаточно свободного места для спавна |
| Торговые автоматы не появляются | Включите нужные торговые автоматы в секции `Vending machines` конфигурации |
| Ошибка при покупке за Economics | Проверьте, что плагин Economics установлен и название валюты указано корректно |
| Отзыв не работает | Убедитесь, что у транспорта включена опция `Can recall?` и у игрока есть право `vehiclebuy.recall` |
| Подбор молотком не работает | Проверьте, что `Pickup type` установлен в `1` (молоток) и `Can pickup?` включен |
