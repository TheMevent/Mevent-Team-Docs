# Battlepass

## Что делает этот плагин?

**Battlepass** позволяет вашему серверу предложить игрокам систему боевого пропуска с ежедневными заданиями, наградами и кейсами. Игроки выполняют разнообразные миссии (добыча ресурсов, убийства, крафт, рыбалка, рейды и многое другое), зарабатывают внутриигровую валюту и тратят её на открытие кейсов с ценными предметами. Все награды хранятся во встроенном инвентаре плагина, откуда игрок может забрать их в любой удобный момент.

## Ключевые возможности

- Ежедневные задания (миссии) с автоматическим обновлением каждые 24 часа
- Персональный «вызов дня» для каждого игрока
- Система из двух валют с поддержкой сторонних экономических плагинов (Economics, ServerRewards и др.)
- Кейсы с настраиваемыми предметами и шансами выпадения
- Встроенный инвентарь для хранения выигранных наград
- VIP-множители на заработок валюты
- Более 25 типов миссий: добыча, убийства, крафт, рыбалка, лут ящиков, взлом закрытых контейнеров, переработка, покупки у NPC и участие в ивентах
- Красивый пользовательский интерфейс с полной настройкой
- Поддержка интеграции с Notify, UINotify, NoEscape
- Автоматический сброс данных при вайпе (настраивается)
- Логирование действий игроков в файл и консоль

## Требования

| Зависимость | Обязательно? | Описание |
|---|---|---|
| ImageLibrary | Да | Загрузка и отображение изображений в интерфейсе |
| Notify / UINotify | Нет | Всплывающие уведомления для игроков |
| NoEscape | Нет | Блокировка действий в инвентаре Battlepass во время рейда |
| Economics / ServerRewards | Нет | Внешняя экономическая система (вместо встроенной) |
| PlayerDatabase | Нет | Альтернативное хранение данных игроков |

## Быстрый старт

1. Установите плагин **ImageLibrary** на сервер, если он ещё не установлен.
2. Поместите файл `Battlepass.cs` в папку `oxide/plugins` вашего сервера.
3. Перезагрузите сервер или выполните команду `o.reload Battlepass`.
4. Откройте файл конфигурации `oxide/config/Battlepass.json` и настройте миссии, кейсы и награды под свой сервер.
5. Проверьте работу плагина, набрав в игровом чате `/pass`.

## Права доступа

| Право | Кому нужно | Что открывает |
|---|---|---|
| `battlepass.use` | Игрокам | Доступ к интерфейсу боевого пропуска |
| `battlepass.vip` | VIP-игрокам | Доступ к привилегированным кейсам и второй валюте (настраивается) |
| `battlepass.premium` | Премиум-игрокам | Повышенный множитель заработка валюты (x3 по умолчанию) |

:::tip Настройка VIP-множителей
Множители заработка первой валюты привязаны к правам доступа. По умолчанию `battlepass.vip` даёт x2, а `battlepass.premium` даёт x3. Вы можете изменить их или добавить свои в разделе конфигурации «Currency 1 > Rates for permissions».
:::

## Команды игроков

| Команда | Что делает | Кто может использовать |
|---|---|---|
| `/pass` | Открывает главное меню боевого пропуска | Все игроки с правом `battlepass.use` |
| `/passmissions` или `/pmissions` | Открывает страницу миссий напрямую | Все игроки с правом `battlepass.use` |
| `/passcase` | Открывает страницу кейсов напрямую | Все игроки с правом `battlepass.use` |
| `/passinv` | Открывает инвентарь наград напрямую | Все игроки с правом `battlepass.use` |

**Пример:** Игрок вводит `/pass`, видит текущие миссии дня и свой прогресс. После выполнения задания на экране появляется уведомление, а заработанная валюта зачисляется на баланс.

## Команды администратора

| Команда | Что делает | Кто может использовать |
|---|---|---|
| `battlepass.wipemissions` | Сбрасывает все миссии и генерирует новые | Только из консоли сервера (RCON) |
| `addfirstcurrency [steamid] [кол-во]` | Начисляет первую валюту игроку | Только администраторы |
| `addsecondcurrency [steamid] [кол-во]` | Начисляет вторую валюту игроку | Только администраторы |
| `battlepass.convert.olddata` | Конвертирует данные из старого формата | Только из консоли сервера (RCON) |

**Пример:**
```
addfirstcurrency 76561198012345678 5000
```
Начислит 5000 единиц первой валюты игроку с указанным Steam ID.

## Устранение неполадок

### Интерфейс не открывается
- **Симптом:** Игрок вводит `/pass`, но ничего не происходит.
- **Причина:** Не установлен ImageLibrary, или у игрока нет права `battlepass.use`.
- **Решение:** Убедитесь, что ImageLibrary установлен и загружен. Выдайте право: `o.grant user ИмяИгрока battlepass.use`.

### В консоли ошибка «IMAGE LIBRARY IS NOT INSTALLED»
- **Симптом:** В консоли сервера повторяется ошибка об отсутствии ImageLibrary.
- **Причина:** Плагин ImageLibrary не установлен или не загрузился.
- **Решение:** Установите ImageLibrary и перезагрузите Battlepass.

### Миссии не обновляются
- **Симптом:** У всех игроков одни и те же задания уже несколько дней.
- **Причина:** Таймер обновления миссий мог сбиться.
- **Решение:** Выполните `battlepass.wipemissions` из серверной консоли, чтобы принудительно обновить миссии.

### Игрок не может забрать предмет из инвентаря
- **Симптом:** При нажатии на предмет в инвентаре появляется сообщение о блокировке.
- **Причина:** Активна блокировка после вайпа (Cooldown After Wipe) или блокировка NoEscape.
- **Решение:** Дождитесь окончания блокировки, либо отключите её в конфигурации.

### Валюта не начисляется
- **Симптом:** Игрок выполняет задание, но баланс не меняется.
- **Причина:** Неправильно настроена интеграция с внешним экономическим плагином.
- **Решение:** Проверьте настройки «Currency 1» в конфигурации. Если используете встроенную систему, убедитесь что параметр «Use embedded system?» включён.

### Кейсы не открываются (ошибка «Недостаточно монет»)
- **Симптом:** Игрок нажимает «Открыть», но видит сообщение о нехватке средств.
- **Причина:** У игрока недостаточно валюты на балансе.
- **Решение:** Убедитесь, что стоимость кейсов в конфигурации адекватна экономике вашего сервера.
