# Multi Events

## Что делает этот плагин?

**Multi Events** позволяет вашему серверу отображать красивый интерфейс для всех серверных ивентов с полной кастомизацией UI, встроенным инвентарём наград и таблицей лидеров. Это основной фреймворк, объединяющий все ивенты сервера в единую визуальную систему с уведомлениями, баннерами и автоматическим запуском случайных событий по таймеру.

## Ключевые возможности

- **Единый интерфейс ивентов** — все серверные ивенты собраны в одном красивом меню
- **Полная кастомизация UI** — встроенный визуальный редактор для изменения баннеров, фонов, кнопок и текстов
- **Инвентарь наград** — игроки могут просматривать полученные награды за участие в ивентах
- **Таблица лидеров** — рейтинг игроков по результатам участия в событиях
- **Уведомления** — всплывающие уведомления при запуске ивентов (интеграция с Notify / UINotify)
- **Автозапуск ивентов** — случайный запуск событий через настраиваемый таймер
- **ServerPanel интеграция** — отображение ивентов через модуль ServerPanel
- **Мультиязычность** — встроенная поддержка английского и русского языков

## Требования

| Требование | Описание |
|---|---|
| Oxide / Carbon | Фреймворк для модификаций сервера |
| ImageLibrary | Обязательно для Oxide (не требуется для Carbon) |

### Опциональные зависимости

| Плагин | Описание |
|---|---|
| Notify / UINotify | Всплывающие уведомления при старте ивентов |
| ServerPanel | Интеграция с панелью сервера |
| MultiEventsInventory | Модуль инвентаря наград |
| MultiEventsLeaderboard | Модуль таблицы лидеров |

## Быстрый старт

1. Установите `MultiEvents.cs` в папку `oxide/plugins`
2. Установите **ImageLibrary** (для Oxide)
3. Перезагрузите сервер или выполните `o.reload MultiEvents`
4. Настройте список ивентов в конфигурации — добавьте ваши серверные ивенты
5. Введите `/events` в чате для просмотра панели ивентов

## Права доступа

| Право | Описание |
|---|---|
| `multievents.edit` | Доступ к визуальному редактору UI и управлению ивентами |

:::note Дополнительные права
Если указаны в конфигурации, доступ к инвентарю и таблице лидеров может требовать отдельных прав доступа.
:::

## Команды игроков

| Команда | Описание |
|---|---|
| `/events` | Открыть панель серверных ивентов |

:::tip Пример
Введите `/events` в чате, чтобы увидеть список активных и предстоящих ивентов на сервере, а также информацию о наградах.
:::

## Команды администратора

| Команда | Право | Описание |
|---|---|---|
| `/ShowUiEditor` | `multievents.edit` | Открыть визуальный редактор интерфейса ивентов |

Через визуальный редактор администратор может:
- Изменять позиции и размеры элементов UI
- Настраивать цвета, шрифты и прозрачность
- Редактировать баннеры и фоновые изображения
- Настраивать кнопки и текстовые элементы

## Устранение неполадок

| Проблема | Решение |
|---|---|
| Команда `/events` не работает | Проверьте, что плагин загружен (`o.reload MultiEvents`). Команду можно изменить в конфигурации |
| Изображения не отображаются | Убедитесь, что ImageLibrary установлен и работает. Проверьте URL изображений в конфигурации |
| Уведомления не появляются | Установите плагин Notify или UINotify. Проверьте параметр "Work with Notify?" в конфигурации |
| Ивенты не запускаются автоматически | Проверьте параметр "Enable timer to start random event" и настройте интервалы запуска |
| Таблица лидеров не отображается | Установите модуль MultiEventsLeaderboard и включите его в конфигурации |
| Визуальный редактор недоступен | Проверьте право `multievents.edit` у вашего пользователя |
